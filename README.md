# 老人保健品即時辨識與安全服用監控系統 — 專案說明文件 (README)

本專案是一個利用樹莓派 4B 與 USB 鏡頭（如羅技 C920），打造的適合長輩或照護者使用的保健品即時辨識與資訊顯示網頁系統。系統會在樹莓派上執行，架設輕量 Flask 網頁伺服器，並提供即時影像串流與辨識結果。

---

## 1. 專案檔案結構
* **`requirements.txt`**：專案所需的 Python 依賴套件（如 Flask, opencv-python, pyzbar）。
* **`database.json`**：保健品條碼資料庫，包含條碼編號、品名、建議劑量、服用時間與銀髮用藥安全警語。
* **`app.py`**：Flask 後端主程式。包含背景鏡頭影像擷取、雙重條碼偵測演算法（PyZbar + OpenCV）以及 SSE (Server-Sent Events) 即時推播機制。
* **`templates/index.html`**：網頁前端介面。採用極具質感的深色玻璃擬態設計，並提供鏡頭影像流、掃描線動畫、即時資訊卡片、歷史紀錄與手動模擬輸入工具。

## 2. 如何在 Windows 本機測試與修改

在部署到樹莓派之前，您可以直接在 Windows 電腦上執行與修改此專案。

### 步驟 A：開啟終端機並切換至專案目錄
開啟終端機（PowerShell 或 CMD），執行以下指令切換至此專案資料夾：
```powershell
cd "c:\大學功課\智慧製造\老人保健品辨識"
```

### 步驟 B：安裝依賴套件
在專案目錄下執行安裝指令：
```powershell
pip install -r requirements.txt
```
> 💡 **ZBar 依賴說明**：如果您的 Windows 系統沒有安裝 `zbar` 底層共享庫，啟動時 `app.py` 會自動偵測並提示 Warning，但**系統不會崩潰**。它會自動進入「模擬模式」並載入「虛擬鏡頭畫面」，這能讓您在無鏡頭、無實體藥罐的情況下，依然可以透過網頁底部的模擬按鈕測試前端卡片與即時資料傳輸。

### 步驟 C：啟動應用程式
在專案目錄下執行：
```powershell
python app.py
```

### 步驟 D：開啟瀏覽器
在瀏覽器輸入 `http://127.0.0.1:5000` 即可進入監控網頁。
* 您可以使用網頁下方的 **「模擬測試與手動輸入」** 區塊，點擊 **「🐟 魚油」**、**「🦴 鈣片」** 等按鈕。
* 網頁上的服用指引卡片會隨即以淡入動畫即時更新，且下方會自動將此掃描加入今日歷史紀錄。

## 3. 如何在樹莓派 4B 上部署與執行

當您修改完程式碼並準備部署到樹莓派時，請依照下列步驟進行設定：

### 步驟 A：硬體連接
1. 將 USB 鏡頭（如羅技 C920）插入樹莓派 4B 的 USB 3.0 插槽（藍色孔）。
2. 插上已寫入作業系統的 MicroSD 卡，並連接官方 5V/3A Type-C 電源啟動樹莓派。

### 步驟 B：安裝樹莓派系統套件（Zbar 函式庫）
樹莓派的 Linux 系統需要安裝底層的 Zbar 條碼辨識庫，`pyzbar` 才能正常工作。請在樹莓派的終端機執行：
```bash
sudo apt-get update
sudo apt-get install libzbar0 -y
```

### 步驟 C：安裝 Python 套件並執行
在樹莓派專案目錄下執行：
```bash
pip install -r requirements.txt
python app.py
```
> 💡 **運作原理**：樹莓派啟動後，它會自動讀取 C920 鏡頭，並以 `640x480` 的合適解析度進行背景影像辨識，以節省處理器資源。

### 步驟 D：遠端存取網頁
與樹莓派處於同一個 Wi-Fi 或區域網路的電腦或手機，只需在瀏覽器輸入：
```
http://<樹莓派的IP位址>:5000
```
即可看到完全同步的即時掃描畫面與結果。

## 4. 系統設計技術特點

* **優雅降級機制 (Graceful Fallback)**：`app.py` 內部會自動檢測是否有可用鏡頭以及是否成功載入 `pyzbar`。若鏡頭或函式庫不可用，會自動開啟「虛擬動態格線鏡頭畫面」，防止程式崩潰，方便開發。
* **Server-Sent Events (SSE)**：相較於傳統網頁一直發送請求去問後端「有沒有掃到條碼？」，SSE 讓後端在掃到條碼的瞬間，才**主動推送**資料到前端，節省頻寬與樹莓派資源。
* **雙重掃描防重機制 (Scan Cooldown)**：同一個保健品條碼在掃過後會進入 4 秒的冷卻時間，避免同一個產品在鏡頭前抖動時，網頁前端不斷閃爍、重複跳出通知。

## 5. 期末作品規範對照表 (報告與 Demo 腳本建議)

本專案已完全符合並超越《AI 邊緣運算期末作品規範建議》之要求，您可以參考以下對照表來準備您的口頭報告與 Demo 影片：

### 🎯 最低完成標準與邊緣/雲端分工
* **影像擷取 (Edge)**：透過 OpenCV `cv2.VideoCapture` 從 USB 網路攝影機取得連續影像，並使用 `cv2.flip` 進行鏡像翻轉，提升使用者對準畫面的直覺性。
* **觸發與前處理 (Edge)**：提供手動快門按鈕，使用者對準保健品後點擊按鈕，樹莓派擷取單張畫面進行壓縮，以節省雲端 API 資源與頻寬。
* **影像辨識 (Cloud)**：將影像送往最新世代的 `Gemini 2.5 Flash` 多模態大模型，進行強大的視覺外觀特徵辨識與包裝文字萃取，解析用藥資訊。
* **邊緣部署 (Edge)**：系統完全設計於 Raspberry Pi 4B 上運行 Flask 伺服器，負責 UI 渲染、影像串流、API 路由與邏輯控制。
* **結果呈現 (Edge)**：透過 SSE 技術將 Gemini 分析完的結構化 JSON 資料即時推送至前端 HTML 網頁，顯示辨識出的「標籤 (品名)」、「文字輸出 (服用警告)」。

### 🌟 加分項目實作
* **加分項 1 (顯示推論時間)**：系統已於影像左上角及網頁儀表板即時顯示送往 Gemini 的 **Inference Time (推論毫秒數)**。
* **加分項 3 (將結果存成 CSV 或影像紀錄)**：所有辨識成功的紀錄與效能數據，皆會自動附加儲存至專案目錄下的 `history.csv`，方便後續實驗結果分析。
* **加分項 4 (設計簡單介面或網頁看板)**：已使用 Flask + HTML/CSS 打造高質感「玻璃擬態 (Glassmorphism)」前端儀表板，支援動態載入提示與狀態推播。
